命令行参数

Linux命令行参数解析函数getopt

＃i nclude <unistd.h>

int getopt(int argc, char * const argv[], const char *optstring);

extern char *optarg;
extern int optind, opterr, optopt;
  • char *optarg 当前选项参数字符串(如果有)
  • int optind argv的当前索引值。当getopt()在while循环中使用时，循环结束后，剩下的字串视为操作数，在argv[optind]至argv[argc-1]中可以找到
  • int opterr 这个变量非零时，getopt()函数为“无效选项”和“缺少参数选项，并输出其错误信息
  • int optopt 当发现无效选项字符之时，getopt()函数或返回'?'字符，或返回':'字符，并且optopt包含了所发现的无效选项字符

ILdb

  • po (print object)
  • po:(print object)输出对象摘要
  • p:(print)输出原生数据，例如int，char，double
  • p/x or print/x 以16进制输出字符
  • expression:执行表达式
  • c continue 继续
  • n next 下一步
  • s step
  • frame variable （varname）: 查看变量数据（包括vector和原生数据类型）
  • vector.__begin 表示vector的首元素位置

breakpoint

在程序中插入断点

int main ()
{
    int i = 0;
    while (i < 30)
    {
        i++; // break here
    }
}

//在原文件中含有'break'字符的地方插入断点
(lldb) breakpoint set -p break
//当断点1满足一定条件时暂停
(lldb) br mod -c 'i % 5 == 0' 1
//在第一个端点触发后执行指令
breakpoint command add 1
Enter your debugger command(s).  Type 'DONE' to end.
> p i //打印i数值
> c //继续运行
> DONE

watchpoint

目的：监视某个变量被读取／写入的动作。

//监视数组的p[1430]数值变化
(lldb) watchpoint set variable p[1430]
//监视读取
(lldb) watchpoint set v -w read _abc4
//监视读写
(lldb) watchpoint set v -w read_write _abc3

当触发watchpoint时，程序暂停
