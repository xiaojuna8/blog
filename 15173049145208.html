随机数检测规范

2018/1/30 posted in  Crypt

随机数检测规范游国家密码管理局提出，GM/T 0005—2012 随机性检测规范。没有找到该文件的电子版，但可以参考。

随机性检测

建议每次样本长度不少于106bit，建议显著性水平$\alpha = 0.01$,检测项目共15项.

分别是单比特频率数检测，块内频数检测，扑克检测，重叠子序列检测，游程总数检测，游程分布检测，块内最大‘1’游程检测，二元推导检测，自相关检测，矩阵秩检测，累加和检测，近似熵检测，线性复杂度检测，Maurer通用统计检测，离散傅立叶检测。

美国的随机数检测标准是NIST 800-22，并提供测试工具sts，测试标准当15项测试P值大于0.01且频率超过0.9806时，则通过测试。

下载make后得到assess，运行时

  1. ./assess 1048576设置单样本bit长度
  2. 对话框选择[0]Input File
  3. 对话框选择1所有项目
  4. 对话框选择0默认参数
  5. 对话框输入1000样本数量
  6. 对话框选择1二进制方式
  7. 等待测试完成
  8. 到experiments\AlgorithmTesting目录查看生成报告

一共15项，对应关系参考文章

  1. Frequency(对应中国：单比特频数检测)
  2. BlockFrequency(对应中国：块内频数检测)
  3. CumulativeSums（对应中国：累加和检测）
  4. Runs(对应中国：游程总数)
  5. LongestRun(对应中国：游程分布)
  6. Rank(对应中国：矩阵秩检测)
  7. FFT(对应中国：离散傅立叶检测)
  8. NonOverlappingTemplate
  9. OverlappingTemplate
  10. Universal(对应中国：Maurer通用统计检测)
  11. ApproximateEntropy(对应中国：近似熵检测)
  12. RandomExcursions
  13. RandomExcursionsVariant
  14. Serial(对应中国：重叠子序列检测)
  15. LinearComplexity(对应中国：线性复杂度检测)

随机数发生器检测

建议样本数为1000，将随机数生成器生产数据存储为二进制文件。对1000个文件进行上面提到的15个项目检测，分别记录各项结果。若某一项通过的样本个数超过981时则视为通过该项目，否则未通过此项检测。

若随机数发生器通过所有检测项目，则随机数发生器通过规范检测，否则，未通过规范检测。

Check available entropy in Linux

参考文章

Sometimes servers just have the weirdest SSL problems ever. In some of these situations, the entropy has been drained. Entropy is the measure of the random numbers available from /dev/urandom, and if you run out, you can’t make SSL connections. To check the status of your server’s entropy, just run the following:

# cat /proc/sys/kernel/random/entropy_avail

If it returns anything less than 100-200, you have a problem. Try installing rng-tools, or generating I/O, like large find operations. Linux normally uses keyboard and mouse input to generate entropy on systems without random number generators, and this isn’t very handy for dedicated servers.

在服务器上进行测试，一般返回值在3000以上。linux熵池有4096位。

Wiki 上 /dev/random

参考链接

/dev/random should be suitable for uses that need very high quality randomness such as one-time pad or key generation.

The intent is to serve as a cryptographically secure pseudorandom number generator, delivering output with entropy as large as possible. This is suggested by the authors for use in generating cryptographic keys for high-value or long-term protection.
