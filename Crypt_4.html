Ciphertext stealing

在加密过程中Ciphertext stealing是一种通用的方法。作用是处理原文长度不是Block整数倍情况。使密文和原文长度相同。

EBC + CS

ECB模式，需要原文长度大于block长度。

CBC模式+CS

CBC模式的Ciphertext stealing对原文长度没有要求，结构简单。

个人看法当原文长度短于一个block时，先对iv进行加密，然后并异或原文，得到密文。解密时，先对iv加密，再异或密文，得到原文。这是仿照CFB模式进行加密解密。

CBC加密

  1. 最后一个blcok补0
  2. 按照标准CBC模式加密
  3. 交换最后两个block密文
  4. 截断最后block长度

ISO10126

最后Block必须填充随机byte.例如block是8Byte，最后有4Byte数据，因此需要填充4Byte随机数

... | DD DD DD DD DD DD DD DD | DD DD DD DD 81 A6 23 04 |

PKCS7

最后Block中查几个Byte就填充对应Byte.例如Block是8Byte，最后有4Byte数据，还差4Byte，因此填充0x04.

... | DD DD DD DD DD DD DD DD | DD DD DD DD 04 04 04 04 |

PKCS5

PKCS5是针对Block8Byte情况。原则与PKCS7相同。

ZeroPadding

填充0.

... | DD DD DD DD DD DD DD DD | DD DD DD DD 00 00 00 00 |

分组加密有若干传统加密模式，分别有不同的特点。 不同的加密模式是为了能隐藏原文的模式。

Symmetric ciphers have good performance, so use them on large data sets.

ECB

EBC(Electronic Codebook)是最简单的加密模式，把明文分成若干段，分别独立进行加密。因此该模式可并行加密。

CBC

CBC(Cipher Block Chaining)，这种模式引入初始化向量，目的是让相同的明文块生成不同的密文。将其加入明文第一个block，并将加密后的密文加入下个block的原文。

初始化向量无需保密，但每次加密时都要不可预测。

PBKDF2

PBKDF2(Password-Based Key Derivation Function)是一个用来生成密钥的函数，常用于生成加密的密码。它通过一个伪随机函数，把明文和盐作为输入，通过一系列重复运算，最终生成密钥

DK = PBKDF2(PRF, Password, Salt, c, dkLen)

其中

  • PRF是伪随机函数，一般用HMAC
  • Password是密码
  • Salt盐
  • c迭代次数
  • dkLen期望得到的key长度
  • DK生成的key
DK = T1 || T2 || ... || T_dklen/hlen
Ti = F(Password, Salt, c, i)

hlen是hash函数生成的hash码长度

其中F函数是异或函数

Ti = F(Password, Salt, c, i) = U1 ^ U2 ^...^ Uc
where
U1 = PRF(Password,Salt || INT_32_BE(i))
U2 = PRF(Password,U1)
...
Uc = PRF(Password,Uc-1)

scrypt

scrypt是一个基于密码的key生成方法，其安全性强于PBKDF2

内存消耗

Memory in bytes = (N*r*128)+(r*p*128)

算法描述

  • Passphrase 用户密码
  • Salt 盐
  • N CPU/memory 花费
  • p 并行参数
  • dkLen 期望得到的key长度
  • r 混合循环blocksize
Function scrypt(Passphrase,Salt,N,p,dkLen):
    (B0 ... Bp−1) ← PBKDF2(HMAC_SHA256, Passphrase, Salt, 1, p * MFLen)
    for i = 0 to p-1 do
        Bi ← SMix(Bi,N)
    end for
    Output ← PBKDF2(HMAC_SHA256, Passphrase, B0 || B1 ... Bp−1, 1, dkLen)

MFLen ？ = 128 * r

Function SMix(B,N):
    X ← B
    for i = 0 to N − 1 do
       Vi ← X
       X ← BlockMix(X)
    end for
    for i = 0 to N − 1 do
        j ← Integerify(X) mod N
        X ← BlockMix(X ⊕ Vj)
    end for
    Output ← X

Integerify() 是\(\{0,1\}^k \rightarrow \{0,1…2^k-1\}\)

Function BlockMix(B):
(B0, ... , B2r-1) ← B
X ← B2r−1
for i = 0 to 2r − 1 do
X ← H(X ⊕ Bi)
Yi ← X
end for
Output ← (Y0, Y2, ... , Y2r−2, Y1, Y3, ... , Y2r−1)

盐(密码学)

通常情况下字符串经过散列处理后无法还原为原始字符串，但是若原始字符串不够随机，很可能已经被作为破解字典，这样原密码会暴露。

加盐的实现通常是在需要散列的字符串特定位置增加特定字符，打乱原始字符串，使其散列结果发生变化。例如用户密码为x7faqgjw

x7faqgjw
MD5:455e0e5c2bc109deae749e7ce0cdd397

若在用户密码后添加特定字符串

x7faqgjwabcdefghijklmnopqrstuvwxyz
        ^^^^^^^^^^^^^^^^^^^^^^^^^^
MD5:4a1690d5eb6c126ef68606dda68c2f79

或者在字符串某些位置插入字符

x7afabqgcjw
  ^  ^  ^

HMAC

HMAC密钥散列消息认证码，为验证数据完整和身份认证

\[ HMAC (K,m) = H( ( K^\prime \oplus opad ) || H((K^\prime\oplus ipad )||m) )\]

  • H是哈希函数
  • K是原始密码
  • m是消息
  • K‘是填充后密码
  • ||是字符串连结符号
  • \(\oplus\)是异或运算符
  • opad是外部填充(0x5c5c5c...)
  • ipad是内部填充(0x363636...)
  • blocksize是分块大小，对于SHA-256，blocksize = 512bits
 function hmac (key, message) {
    if (length(key) > blocksize) {
        key = hash(key) // keys longer than blocksize are shortened
    }
    if (length(key) < blocksize) {
        // keys shorter than blocksize are zero-padded 
        // Where * is repetition.(where ∥ is concatenation)
        key = key ∥ [0x00 * (blocksize - length(key))] 
    }
    // Where blocksize is that of the underlying hash function
    o_key_pad = [0x5c * blocksize] ⊕ key
    // Where ⊕ is exclusive or (XOR)
    i_key_pad = [0x36 * blocksize] ⊕ key
     // Where ∥ is concatenation
    return hash(o_key_pad ∥ hash(i_key_pad ∥ message))
}
