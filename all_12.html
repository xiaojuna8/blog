Intel random number generator

Intel recently announced the addition of the RDSEED instruction to the Intel® 64 and IA-32 Architectures. An addition to Intel® Secure Key, this new instruction will appear in future generations of Intel processors.

Like RDRAND the RDSEED instruction returns random numbers, and with two instructions that appear to do the same thing it is only natural for the software developer to ask, "Which one should I use?"

The Difference Between RDRAND and RDSEED

short answer

  • if you want to seed another pseudorandom number generator,use RESEED
  • for other purposes, use RDRAND

long answer

Instruction Source NIST Compliance
RDRAND Crypto secure pseudorandom number generator SP 800-90A
RDSEED Non-deterministic random number generator SP 800-90B&C

使用RDSEED可生成两个独立随机数。

使用RDRAND只能生成两个高质量随机数，他们并不独立。

因此说，RDSEED生成的随机数质量高于RDRAND

NIST RBG standard

NIST SP 800-90A : DRGB

NIST SP 800-90B : Entropy source

NIST SP 800-90C : RBG = Entropy source + DRBG

AMD rand

AMD RYZEN and EPYC 处理器中也包含随机数生成器。

mbed TSL

Entropy source，On Linux, /dev/urandom is used by default. On Windows, the CryptGenRandom() function is used
