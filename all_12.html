基于标识的加密

  • 优势
  • 缺点

Identity-based systems allow any party to generate a public key from a known identity value such as an ASCII string.A trusted third party, called the Private Key Generator (PKG), generates the corresponding private keys.

任意组织都可以通过标识生成公钥，可靠的第三方PKG生成相关的私钥。

To operate, the PKG first publishes a master public key, and retains the corresponding master private key (referred to as master key).

PKG公布公钥（a master public key一个主公钥，只有一个），保留主私钥。

Given the master public key, any party can compute a public key corresponding to the identity by combining the master public key with the identity value.

根据主公钥，任何人都可以计算出与ID有关的公钥。

To obtain a corresponding private key, the party authorized to use the identity ID contacts the PKG, which uses the master private key to generate the private key for identity ID.

为了获取相关私钥，经授权使用该ID的人联系PKG获取私钥。

优势

once issued, keys are always valid

密钥一次分发，永久有效。

缺点

If a Private Key Generator (PKG) is compromised, all messages protected over the entire lifetime of the public-private key pair used by that server are also compromised.

一旦PKG的主私钥危险，所有发送过的公钥-私钥对都危险。

To limit the exposure due to a compromised server, the master private-public key pair could be updated with a new independent key pair.

为了预防这种情况，主私钥可以升级。

However, this introduces a key-management problem where all users must have the most recent public key for the server.

但是这又会导致其它密钥管理方面的问题。

Because the Private Key Generator (PKG) generates private keys for users, it may decrypt and/or sign any message without authorization.

PKG有能力伪造用户进行加密／签名。

Makefile

自动化编译工具

功能

  1. 将编译指令记录到一个文件中，方便调用和修改
  2. 可以添加编译选项，在不改动代码情况下控制编译

用法

编写好makefile文件后，在Terminal中敲make指令，就按照makefile中的指令进行编译。

语法

每条语句分为三个部分

  1. 目标名称
  2. 依赖项，依赖项可以是文件也可以是其他目标
  3. 指令
first: main.cpp fun.cpp
    gcc -o test main.cpp fun.cpp

makefile只完成第一个目标。可以在第一个目标的依赖项中加入其它目标，实现先后编译。

gcc是C编译器，g++是C++编译器。

GCC选项

g++是C++编译器

-D是定义编译选项，例如-DDEBUG为定义DEBUG宏

-std是指定C++版本，例如-std=c++11

-O是编译优化选项，例如-O3

杂项

注释，在makefile中以“#”开始的行为注释内容。

makefile每一句指令默认都会在Terminal中显示，若不想在Terminal中显示可以在指令开头加上“@”.

若想在Terminal中显示一句话可以用echo指令，例如@echo 开始编译xx模块

要删除中间生成的.o文件可以用指令rm *.o
