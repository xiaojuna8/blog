ILdb

  • po (print object)
  • po:(print object)输出对象摘要
  • p:(print)输出原生数据，例如int，char，double
  • p/x or print/x 以16进制输出字符
  • expression:执行表达式
  • c continue 继续
  • n next 下一步
  • s step
  • frame variable （varname）: 查看变量数据（包括vector和原生数据类型）
  • vector.__begin 表示vector的首元素位置

breakpoint

在程序中插入断点

int main ()
{
    int i = 0;
    while (i < 30)
    {
        i++; // break here
    }
}

//在原文件中含有'break'字符的地方插入断点
(lldb) breakpoint set -p break
//当断点1满足一定条件时暂停
(lldb) br mod -c 'i % 5 == 0' 1
//在第一个端点触发后执行指令
breakpoint command add 1
Enter your debugger command(s).  Type 'DONE' to end.
> p i //打印i数值
> c //继续运行
> DONE

watchpoint

目的：监视某个变量被读取／写入的动作。

//监视数组的p[1430]数值变化
(lldb) watchpoint set variable p[1430]
//监视读取
(lldb) watchpoint set v -w read _abc4
//监视读写
(lldb) watchpoint set v -w read_write _abc3

当触发watchpoint时，程序暂停

git tag

标签可以将提交标记为便于阅读和发行的版本号。例如可以在某个提交后添加标签为v1.0，将Terminal进入到本地仓库目录。

  • git tag 列出所有标签
  • git tag -a v1.4.2 -m 'my version 1.4.2'创建标签和message
  • git show v1.4.3显示标签
  • git push origin --tags标签推送到版本库
  • git push --delete origin YOUR-TAG-NAME删除某个标签
