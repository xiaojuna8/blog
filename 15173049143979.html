Best practice? In order of preference, use:

  1. scrypt
  2. bcrypt
  3. Argon2
  4. sha512crypt
  5. sha256crypt
  6. PBKDF2

Do not use:

  1. MD5
  2. md5crypt
  3. UNIX crypt(3)
  4. SHA-1/2/3
  5. Skein
  6. BLAKE2
  7. Any general purpose hashing function.
  8. Any encryption algorithm.
  9. Your own design.
  10. Plaintext

PBKDF2

PBKDF2(Password-Based Key Derivation Function)是一个用来生成密钥的函数，常用于生成加密的密码。它通过一个伪随机函数，把明文和盐作为输入，通过一系列重复运算，最终生成密钥

DK = PBKDF2(PRF, Password, Salt, c, dkLen)

其中

  • PRF是伪随机函数，一般用HMAC
  • Password是密码
  • Salt盐
  • c迭代次数
  • dkLen期望得到的key长度
  • DK生成的key
DK = T1 || T2 || ... || T_dklen/hlen
Ti = F(Password, Salt, c, i)

hlen是hash函数生成的hash码长度

其中F函数是异或函数

Ti = F(Password, Salt, c, i) = U1 ^ U2 ^...^ Uc
where
U1 = PRF(Password,Salt || INT_32_BE(i))
U2 = PRF(Password,U1)
...
Uc = PRF(Password,Uc-1)

scrypt

scrypt是一个基于密码的key生成方法，其安全性强于PBKDF2

内存消耗

Memory in bytes = (N*r*128)+(r*p*128)

算法描述

  • Passphrase 用户密码
  • Salt 盐
  • N CPU/memory 花费
  • p 并行参数
  • dkLen 期望得到的key长度
  • r 混合循环blocksize
Function scrypt(Passphrase,Salt,N,p,dkLen):
    (B0 ... Bp−1) ← PBKDF2(HMAC_SHA256, Passphrase, Salt, 1, p * MFLen)
    for i = 0 to p-1 do
        Bi ← SMix(Bi,N)
    end for
    Output ← PBKDF2(HMAC_SHA256, Passphrase, B0 || B1 ... Bp−1, 1, dkLen)

MFLen ？ = 128 * r

Function SMix(B,N):
    X ← B
    for i = 0 to N − 1 do
       Vi ← X
       X ← BlockMix(X)
    end for
    for i = 0 to N − 1 do
        j ← Integerify(X) mod N
        X ← BlockMix(X ⊕ Vj)
    end for
    Output ← X

Integerify() 是\(\{0,1\}^k \rightarrow \{0,1…2^k-1\}\)

Function BlockMix(B):
(B0, ... , B2r-1) ← B
X ← B2r−1
for i = 0 to 2r − 1 do
X ← H(X ⊕ Bi)
Yi ← X
end for
Output ← (Y0, Y2, ... , Y2r−2, Y1, Y3, ... , Y2r−1)

盐(密码学)

通常情况下字符串经过散列处理后无法还原为原始字符串，但是若原始字符串不够随机，很可能已经被作为破解字典，这样原密码会暴露。

加盐的实现通常是在需要散列的字符串特定位置增加特定字符，打乱原始字符串，使其散列结果发生变化。例如用户密码为x7faqgjw

x7faqgjw
MD5:455e0e5c2bc109deae749e7ce0cdd397

若在用户密码后添加特定字符串

x7faqgjwabcdefghijklmnopqrstuvwxyz
        ^^^^^^^^^^^^^^^^^^^^^^^^^^
MD5:4a1690d5eb6c126ef68606dda68c2f79

或者在字符串某些位置插入字符

x7afabqgcjw
  ^  ^  ^

HMAC

HMAC密钥散列消息认证码，为验证数据完整和身份认证

\[ HMAC (K,m) = H( ( K^\prime \oplus opad ) || H((K^\prime\oplus ipad )||m) )\]

  • H是哈希函数
  • K是原始密码
  • m是消息
  • K‘是填充后密码
  • ||是字符串连结符号
  • \(\oplus\)是异或运算符
  • opad是外部填充(0x5c5c5c...)
  • ipad是内部填充(0x363636...)
  • blocksize是分块大小，对于SHA-256，blocksize = 512bits
 function hmac (key, message) {
    if (length(key) > blocksize) {
        key = hash(key) // keys longer than blocksize are shortened
    }
    if (length(key) < blocksize) {
        // keys shorter than blocksize are zero-padded 
        // Where * is repetition.(where ∥ is concatenation)
        key = key ∥ [0x00 * (blocksize - length(key))] 
    }
    // Where blocksize is that of the underlying hash function
    o_key_pad = [0x5c * blocksize] ⊕ key
    // Where ⊕ is exclusive or (XOR)
    i_key_pad = [0x36 * blocksize] ⊕ key
     // Where ∥ is concatenation
    return hash(o_key_pad ∥ hash(i_key_pad ∥ message))
}
