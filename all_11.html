AES是美国政府指定的商业分组加密方案，于2001年公布。

AES中概念

  • word 32bit data or 4Byte
  • Nb Number of columns, Nb = 4
  • Nk Number of words for key
  • Nr Number of rounds

AES的三种模式

AES的blocksize是16Byte，密钥分为128/192/256三种长度。

  • 128bit 10轮加密
  • 192bit 12轮加密
  • 256bit 14轮加密
Key Lenght(Nk words) Block Size (Nb words) Number of Rounds(Nr)
AES128 4 4 10
AES192 6 4 12
AES256 8 4 14

通过观察可得Nr = Nk + 6

密钥扩展

AES加密根据密钥长度不同分别有不同轮数的加密，在对每个block每轮加密所使用的轮密钥都是从初始密钥生成。密钥扩展是串行的，每个word生成都依赖于前一个word

测试1

在Intel网站2014年有篇文章，其中在RAM测试提到Ivy Bridge加密速度为581.3MB／s，Haswell加密速度663.8MB／s。这与架构处理从8周期减少到7周期基本吻合。(指令\(7/8 = 87.5\%\)，实测 \(581.3/663.8 \approx 87.6\%\))注意到测试中所用Haswell处理器频率稍低，反而加密速度更快。

(XTS) modes with both 128- and 256-bit keys. Note that for XTS mode, only half the key is used, so XTS-512 essentially utilizes a 256-bit key.

Crypto++

Crypto++ 5.6.5 Benchmarks中，The host CPU is a 6th generation Skylake, frequency is 3.14GHz.测试AES/128/CBC模式速度为1073MiB／s，AES/128/CTR模式速度为4499MiB/s，所以CTR并行模式可以大幅加速加密解密。曾经 "AES finalists"，Sepent/128/CTR只有87MiB／s，Twofish／128/CTR只有181MiB／s，而Camellia/128/CTR只有153MiB／s。所以可以看出AES-NI对速度提升影响很大，同时说明实现AES128强度加密用软件，其加密速度在百兆多以上，但很难超过200MiB／s

ECC性能

原文

更适合移动互联网

ECC加密算法的密钥长度很短(256位)，意味着占用更少的存储空间，更低的CPU开销和占用更少的带宽。随着越来越多的用户使用移动设备来完成各种网上活动，ECC加密算法为移动互联网安全提供更好的客户体验。

更好的安全性

ECC加密算法的密钥长度很短(256位)，意味着占用更少的存储空间，更低的CPU开销和占用更少的带宽。随着越来越多的用户使用移动设备来完成各种网上活动，ECC加密算法为移动互联网安全提供更好的客户体验。

更好的性能

ECC加密算法需要较短的密钥长度来提供更好的安全，例如，256位的ECC密钥加密强度等同于3072位RSA密钥的水平(目前普通使用的RSA密钥长度是2048位)。其结果是你以更低的计算能力代价得到了更高的安全性。经国外有关权威机构测试，在Apache和IIS服务器采用ECC算法，Web服务器响应时间比RSA快十几倍。

ECC特点

ECC加密算法自1985年提出，因其工程应用中难度较高，到2005年才在各种操作系统中获得广泛支持，在全球安全市场需求的刺激下，ECC算法将逐步取代RSA算法，成为主流加密算法。目前，全球各大CA都已经陆续开始为用户签发采用ECC加密算法的各种证书。
