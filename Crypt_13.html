分组加密的加密模式

  • ECB
  • CBC
  • CFB
  • CTR
  • GCM
  • 加密方式总结

分组加密有若干传统加密模式，分别有不同的特点。 不同的加密模式是为了能隐藏原文的模式。

Symmetric ciphers have good performance, so use them on large data sets.

ECB

EBC(Electronic Codebook)是最简单的加密模式，把明文分成若干段，分别独立进行加密。因此该模式可并行加密。

并行
加密 是
解密 是
随机读取 是

这样相同的原文会被加密为相同的密文，在某些场合不能使用。

CBC

CBC(Cipher Block Chaining)，这种模式引入初始化向量，目的是让相同的明文块生成不同的密文。将其加入明文第一个block，并将加密后的密文加入下个block的原文。

初始化向量无需保密，但每次加密时都要不可预测。

加密 否
解密 是
随机读取 是

CBC模式是最常用的模式，缺点是串行处理。

CFB

CFB(Cipher Feedback)，使用注意无论加密还是解密，都用的是加密函数。

解密 是
随机读取 是

CTR

GCM

GCM模式方便进行并行处理，GCM往往与AES一起使用。GCM可认为是一种特殊的CTR模式(规定了counter位数和递增方式，但GCM并未规定couter初始值)。GCM可适应非blockSize整数倍的原文长度,但加密有64GB(假定blockSize是16B)文件大小限制。

其中inc函数定义为

\[inc_s (X) = MSB_{len(X)-s}(X) || [int(LSB_s(X))+1\hspace{2mm} mod \hspace{2mm} 2^s]_s\]

In other words, the function increments the right-most s bits of the string, regarded as the binary representation of an integer, modulo 2s; the remaining, left-most len(X)-s bits remain unchanged.

特别的，在GCM中规定s = 32,该函数退化为，前96bit不变，最后32bit当作uint32进行+1操作。

加密方式总结

ECB方式是安全性最差的方式，不推荐。
